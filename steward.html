Executes a real MySQL BEGIN … COMMIT transaction that:
1. Validates the session and driver exist
2. Inserts a new row into INCIDENT
3. Immediately inserts a linked row into PENALTY (IncidentID as FK)
4. COMMITs both inserts atomically — or ROLLBACKs both if anything fails.

Tick "Simulate failure" to crash the transaction after step 3 — the INCIDENT insert is rolled back too, proving true atomicity.

How This Demonstrates N:N
INCIDENT table records the event (session, lap, severity).
PENALTY table is a junction between INCIDENT and DRIVER — creating an N:N relationship: one incident can result in penalties for multiple drivers, and one driver accumulates penalties from many incidents.

The transaction ensures both rows exist together or neither does — the database never ends up with an orphaned INCIDENT or a dangling PENALTY foreign key.
