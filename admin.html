How MySQL Lock Contention Works
SELECT … FOR UPDATE
Acquires an exclusive row-level lock. Any other transaction trying the same row with FOR UPDATE will block until the first commits or rolls back — MySQL's built-in serialization mechanism.
InnoDB MVCC
InnoDB uses Multiversion Concurrency Control. Readers don't block writers for snapshot reads, but FOR UPDATE opts into explicit locking. With REPEATABLE READ isolation, each transaction sees a consistent snapshot from when it began.
